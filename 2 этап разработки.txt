# Документация проекта: Merchberry ERP (Версия 1.0 — Предрелизная)

## 1. Концепция и видение продукта (Vision)
**Merchberry ERP** — это легковесная, быстрая и защищенная система управления предприятием (ERP), разработанная специально для учета товаров, финансов и взаимоотношений с авторами мерча и арендаторами полок.

Система решает главную боль розничных точек: автоматизирует расчет комиссий, избавляет от потери данных благодаря системе двухуровневой синхронизации (SKU + Наименование) и предоставляет удобные ролевые интерфейсы для Администратора и Авторов.

---

## 2. Технологический стек
**Архитектура:** Клиент-серверная (SPA + REST API)

**Backend (Серверная часть):**
* Язык: Python 3.12+
* Фреймворк: FastAPI
* База данных: Текущая — SQLite, Целевая — PostgreSQL (через ORM SQLModel)
* Авторизация: JWT-токены, хэширование паролей (`bcrypt`)
* Интеграции: `requests` + `hashlib` для MD5-подписи запросов к API Бизнес.Ру
* Фоновые задачи: APScheduler (планировщик)

**Frontend (Клиентская часть):**
* Сборщик: Vite
* Библиотека: React (TypeScript)
* Стилизация: Tailwind CSS v4
* Маршрутизация: React Router DOM v6
* HTTP-клиент: Axios (с настроенными Interceptors для токенов)
* Иконки: Lucide React

## 3. Обновленная архитектура базы данных (Модели)

**1. Модель `User` (Авторы и Администратор)**
* `id`, `username`, `hashed_password`, `role` ("admin" / "seller")
* `full_name`, `phone` — контактные данные и реквизиты.
* `commission_percent` — процент комиссии магазина (индивидуальный для каждого).
* `balance` — текущий баланс автора к выплате.
* `notes` — приватные комментарии (видит только админ, полезно для сохранения реквизитов банков).
* `is_active` — статус аккаунта (False = доступ закрыт, альтернатива удалению).
* `created_at` — дата создания (используется для ограничения полного удаления 72 часами).

**2. Модель `Product` (Склад и Товары)**
* `id`, `sku` (Артикул Бизнес.Ру), `name` (Уникальное наименование).
* `base_price` (справочная цена), `stock` (остаток).
* `seller_id` (Связь с автором). *Если null — товар находится в "Нераспределенных".*
* `is_obsolete` — флаг отправки товара в архив (вместо физического удаления).

**3. Модель `Transaction` (Финансовые операции)**
* `id`, `type` ("sale", "payout", "rent", "correction").
* `amount` (Сумма чистой прибыли или списания), `commission_amount` (Сумма удержанной комиссии).
* `date`, `comment` (Ссылка на чек Бизнес.Ру или период аренды).
* `seller_id` (Чья операция), `product_identifier` (Название проданного товара).

## 6. Текущие проблемы и зоны риска

1. **Слепая зона API:** Пока логика синхронизации опирается на моковые (тестовые) данные и документацию. В реальном ответе Бизнес.Ру структура вложенности (массивы `goods`) может слегка отличаться от документации, что потребует оперативной отладки при первом запуске.
2. **Восстановление паролей:** В системе нет автоматического сброса паролей через Email. В случае утери доступа, автор не может восстановить его сам — требуется прямое обращение к администратору, который сгенерирует новый пароль в панели. На данном этапе это приемлемо, но увеличивает административную нагрузку.
3. **Рассинхронизация остатков:** В данный момент при синхронизации продаж мы не списываем остаток (`stock`) в локальной базе `Product`, так как мастер-системой по остаткам является Бизнес.Ру. Из-за этого локальные остатки на складе в ERP могут быть неактуальными до загрузки свежего CSV-файла.

## 7. Идеи для развития и усиления (Без усложнения ядра)

* **Экспорт таблиц:** Добавить маленькую кнопку "Скачать CSV" на странице Финансов. Это позволит администратору выгружать все выплаты за месяц в Excel одним кликом для удобного перевода денег через банковские приложения.
* **Главный Дашборд (Статистика):** На странице `/` вывести 2-3 визуальных графика (с помощью библиотеки Recharts). Например: столбчатая диаграмма "Выручка по дням" и список "Топ-5 продаваемых товаров".
* **Уведомления в Telegram:** В перспективе систему можно легко связать с простым Telegram-ботом. Когда админ нажимает кнопку "Выплата", бот может автоматически присылать автору сообщение: *"Вам начислена выплата X руб. Текущий баланс: 0 руб"*.
* **История изменений склада:** Добавить простую лог-таблицу, фиксирующую, кто и когда изменил цену или остаток товара, чтобы исключить человеческий фактор при инлайн-редактировании.
